This guide walks through the complete checkout process in the Flutter Billing App, from scanning items to printing receipts.

## Overview

The checkout workflow consists of three main stages:

1. **Scanning Items** - Add products to cart via barcode scanning
2. **Review Order** - Verify cart contents and total
3. **Print Receipt** - Generate and print customer receipt

## Stage 1: Scanning Items

### Starting a New Order

The home page features a live barcode scanner occupying the top 40% of the screen:

<Steps>

  </Step>
  
  <Step title="Position barcode">

<Note>

### Barcode Detection Logic

Implementation from `lib/features/billing/presentation/pages/home_page.dart:37`:

```dart
void _onDetect(BarcodeCapture capture) async {
  final List<Barcode> barcodes = capture.barcodes;
  final now = DateTime.now();

  for (final barcode in barcodes) {
    if (barcode.rawValue != null) {
      final rawValue = barcode.rawValue!;

      // Cooldown logic: 2 seconds per identical barcode
      if (_lastScanTimes.containsKey(rawValue)) {
        final lastScan = _lastScanTimes[rawValue]!;
        if (now.difference(lastScan).inSeconds < 2) {
          continue;
        }
      }

      _lastScanTimes[rawValue] = now;

      // Vibrate for feedback
      final hasVibrator = await Vibration.hasVibrator();
      if (hasVibrator == true) {
        Vibration.vibrate();
      }

      if (mounted) {
        context.read<BillingBloc>().add(ScanBarcodeEvent(rawValue));
      }
      break;
    }
  }
}
```

### Adding Products to Cart

When a barcode is scanned, the billing logic retrieves and adds the product (`lib/features/billing/presentation/bloc/billing_bloc.dart:25`):

```dart
Future<void> _onScanBarcode(
    ScanBarcodeEvent event, Emitter<BillingState> emit) async {
  final result = await getProductByBarcodeUseCase(event.barcode);
  result.fold(
    (failure) =>
        emit(state.copyWith(error: 'Product not found: ${event.barcode}')),
    (product) {
      add(AddProductToCartEvent(product));
    },
  );
}
```

<Warning>
If a scanned barcode doesn't match any product in your inventory, you'll see a red error message: "Product not found: [barcode]". Add the product to inventory first before scanning.
</Warning>

### Quantity Update Implementation

Quantity adjustment logic (`lib/features/billing/presentation/bloc/billing_bloc.dart:65`):

```dart
void _onUpdateQuantity(
    UpdateQuantityEvent event, Emitter<BillingState> emit) {
  if (event.quantity <= 0) {
    add(RemoveProductFromCartEvent(event.productId));
    return;
  }

  final index = state.cartItems
      .indexWhere((item) => item.product.id == event.productId);
  if (index >= 0) {
    final items = List<CartItem>.from(state.cartItems);
    items[index] = items[index].copyWith(quantity: event.quantity);
    emit(state.copyWith(cartItems: items));
  }
}
```

<Info>
If you scan the same product multiple times, the app automatically increments the quantity instead of creating duplicate cart entries.
</Info>

### Auto-Reconnection

If the printer connection was lost, the app attempts automatic reconnection (`lib/features/billing/presentation/bloc/billing_bloc.dart:85`):

```dart
Future<void> _onPrintReceipt(
    PrintReceiptEvent event, Emitter<BillingState> emit) async {
  final printerHelper = PrinterHelper();

  if (!printerHelper.isConnected) {
    final savedMac = HiveDatabase.settingsBox.get('printer_mac');
    if (savedMac != null) {
      final connected = await printerHelper.connect(savedMac);
      if (!connected) {
        emit(state.copyWith(
            error: 'Failed to auto-connect to printer!', clearError: false));
        return;
      }
    } else {
      emit(state.copyWith(
          error: 'Printer not connected & no saved printer found!',
          clearError: false));
      return;
    }
  }

  // Continue with printing...
}
```

### Receipt Format

The printed receipt includes (`lib/core/utils/printer_helper.dart:106`):

1. **Header** (centered, bold, large)
   - Shop name

2. **Shop Details** (centered)
   - Address line 1
   - Address line 2 (if provided)
   - Phone number
   - Date and time (e.g., "14-03-2026 02:30 PM")

3. **Items Table** (left-aligned)
   ```
   Item            Price   Total
   --------------------------------
   2x Basmati Rice 450.00  900.00
   1x Tata Salt    20.00   20.00
   ```

4. **Total** (right-aligned, bold)
   ```
   TOTAL: 920.0
   ```

5. **Footer** (centered)
   - Custom footer text (e.g., "Thank you, Visit again!!!")

### ESC/POS Print Commands

Receipt generation uses ESC/POS commands for formatting (`lib/core/utils/printer_helper.dart:106`):

```dart
Future<void> printReceipt({
  required String shopName,
  required String address1,
  required String address2,
  required String phone,
  required List<Map<String, dynamic>> items,
  required double total,
  required String footer,
}) async {
  if (!_isConnected) return;

  List<int> bytes = [];

  // Init
  bytes += EscPos.init;

  // Shop Name (Center, Bold, Large)
  bytes += EscPos.alignCenter;
  bytes += EscPos.boldOn;
  bytes += EscPos.textLarge;
  bytes += _textToBytes(shopName);
  bytes += EscPos.lineFeed;

  // ... (address, phone, items, total, footer)

  await PrintBluetoothThermal.writeBytes(bytes);
}
```

<Note>
ESC/POS commands control text alignment, size, and styling on thermal printers. The app uses standard commands compatible with most 58mm and 80mm printers.
</Note>

## Troubleshooting

### "Product not found" error

**Cause:** Scanned barcode doesn't exist in inventory.

**Solution:**
1. Go to Settings → Products
2. Add the product with the correct barcode
3. Return to home and rescan

### Printer not responding

**Solutions:**
1. Check printer is powered on and has paper
2. Go to Settings → Hardware → Print Device
3. Tap **Refresh** to reconnect
4. If failed, tap Settings gear → re-pair printer → Refresh

### Receipt prints but is blank

**Cause:** Paper roll installed backwards.

**Solution:**
1. Open printer cover
2. Flip the paper roll (thermal side should face the print head)
3. Reprint receipt

### Scanner not detecting barcodes

**Solutions:**
1. Clean camera lens
2. Improve lighting conditions
3. Hold phone steady and ensure barcode is in focus
4. Try toggling flashlight on
5. Verify camera permissions are granted

### Cart shows wrong total

**Cause:** Price updates didn't affect current cart.

**Solution:**
1. Go back to home page (clears cart)
2. Rescan items with updated prices

## Best Practices
